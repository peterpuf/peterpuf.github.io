1.pip3 install xxx显示error: externally-managed-environment：

树莓派5新操作系统基于代号为“Bookworm”的Debian 12，使用pip3 install时显示error: externally-managed-environment：

error: externally-managed-environment
 
× This environment is externally managed
╰─> To install Python packages system-wide, try apt install
    python3-xyz, where xyz is the package you are trying to
    install.
    
    If you wish to install a non-Debian-packaged Python package,
    create a virtual environment using python3 -m venv path/to/venv.
    Then use path/to/venv/bin/python and path/to/venv/bin/pip. Make
    sure you have python3-full installed.
    
    If you wish to install a non-Debian packaged Python application,
    it may be easiest to use pipx install xyz, which will manage a
    virtual environment for you. Make sure you have pipx installed.
    
    See /usr/share/doc/python3.11/README.venv for more information.
 
note: If you believe this is a mistake, please contact your Python installation or OS distribution provider. You can override this, at the risk of breaking your Python installation or OS, by passing --break-system-packages.
hint: See PEP 668 for the detailed specification.

如果还是想使用pip3 install的方式安装，解决方案如下：

sudo mv /usr/lib/python3.x/EXTERNALLY-MANAGED /usr/lib/python3.x/EXTERNALLY-MANAGED.bk

注：其中python3.x是你树莓派 5的python实际版本，不知道的可以在terminal中输入python3查看结果，例如我的是python3.11，那么以上命令就是：

sudo mv /usr/lib/python3.11/EXTERNALLY-MANAGED /usr/lib/python3.11/EXTERNALLY-MANAGED.bk

然后不出意外应该就可以进行安装了。

3.Debian 软件源更换（国内提速）

参考链接：清华大学开源软件镜像站

省流：直接将以下代码替换/etc/apt/sources.list中的内容

# 默认注释了源码镜像以提高 apt update 速度，如有需要可自行取消注释
deb https://mirrors.tuna.tsinghua.edu.cn/debian/ bookworm main contrib non-free non-free-firmware
# deb-src https://mirrors.tuna.tsinghua.edu.cn/debian/ bookworm main contrib non-free non-free-firmware

deb https://mirrors.tuna.tsinghua.edu.cn/debian/ bookworm-updates main contrib non-free non-free-firmware
# deb-src https://mirrors.tuna.tsinghua.edu.cn/debian/ bookworm-updates main contrib non-free non-free-firmware

deb https://mirrors.tuna.tsinghua.edu.cn/debian/ bookworm-backports main contrib non-free non-free-firmware
# deb-src https://mirrors.tuna.tsinghua.edu.cn/debian/ bookworm-backports main contrib non-free non-free-firmware

# 以下安全更新软件源包含了官方源与镜像站配置，如有需要可自行修改注释切换
deb https://security.debian.org/debian-security bookworm-security main contrib non-free non-free-firmware
# deb-src https://security.debian.org/debian-security bookworm-security main contrib non-free non-free-firmware

4.cmake安装编译

下载：http://www.cmake.org/files/v3.13/cmake-3.13.0.tar.gz
不同的版本直接替换数字就可。解压后以3.13版本为例：

cd cmake-3.13.0
./configure
make
sudo make install
cmake --version

能查到版本就okk了。

怎么能没有问题那，这里倒不是安装编译错误，而是想安装到指定的位置怎么办，默认应该是安装在/usr/local/bin下面(可以用which cmake查看路径)，那么一般需要手动添加路径到环境变量，编辑~/.bashrc或~/.profile文件，添加以下行：

export PATH=/usr/local/bin:$PATH

刷新环境变量：

source ~/.bashrc

那么如果我想安装到指定位置那？

在运行./configure时，使用--prefix选项来指定安装路径。例如，将CMake安装到/usr，这样cmake可执行文件将位于/usr/bin目录中：

./configure --prefix=/usr

其它步骤还是老样子。
