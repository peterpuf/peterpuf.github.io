### 一、iOS越狱方式

**目前主流的越狱方式分为iOS10-11, 和iOS12-14两大类**

<br />
#### iOS10-11越狱方式
1、通过爱思助手越狱:

将设备连接电脑, 打开爱思助手 -> 工具箱 -> 一键越狱, 按照提示流程操作即可.

<br />
2、手动越狱
iOS10-11版本的设备越狱可以通过在手机的Safari访问[https://meridian.sparkes.zone/](https://meridian.sparkes.zone/)进行越狱, 网站提示很全, 按照流程走即可。越狱成功后设备会重启


<br />
#### iOS12-14越狱方式
1、通过爱思助手越狱: 与低版本设备越狱方式同.

2、下载unc0ver进行越狱
在unc0ver官网[https://unc0ver.dev/](https://unc0ver.dev/)下载最新的安装包, 并使用```iOS-app-singer```[https://github.com/DanTheMan827/ios-app-signer](https://github.com/DanTheMan827/ios-app-signer)进行手动重签名, 签名后安装到手机上, 之后在手机上打开unc0ver按照流程操作即可。

3、通过Crackra1n越狱
在Crackra1n官网[https://checkra.in/](https://checkra.in/)下载安装包, 并安装在Mac电脑上, 随后打开软件, 按照提示进行操作. 


### Tweak插件
在越狱环境下, 我们可以通过Theos编写Tweak的越狱插件, 从而达到对系统、原生App进行hook. 

#### Theos环境安装
```
sudo git clone --recursive https://github.com/theos/theos.git /opt/theos
```
安装完成后, 将Theos加入环境变量.

```
vim ~/.bash_profile 

export THEOS=/opt/theos
export PATH=$PATH:$THEOS
```
随后使用```$THEOS/bin/nic.pl``` 验证是否安装成功.



### 开发Tweak插件

#### 创建Tweak模版
安装好Theos环境后, 就可以使用其中的nic.pl生成一个Tweak插件模版了. 在命令行输入```$THEOS/bin/nic.pl```, 可以在控制台上看到很多类别, 这里选择15, 然后回车

![Tweak](./tweak1.png)

填写项目名称、作者名称、以及需要hook的App包名(绿色框)

![Tweak](./tweak2.png)

这样一个Tweak的模版就创建好了.


#### Tweak项目说明
创建好Tweak模版后, 使用VSCode打开, 可以看到其中包含了4个文件

- control -> Tweak项目说明, 存放一些基本信息
- Makefile -> 打包Tweak项目使用的MakeFile文件, 声明要打包的文件、引入的框架等.
- Tweak.x -> 真实的Hook代码在这里写, 是使用Logos语法写的, 编译后会生成.xm的文件
- TweakDemo.plist -> 插件权限、hook的App包名

![Tweak](./tweak3.png)

在这些文件中, 除了Hook代码的开发之外, 最重要的就是这个MakeFile了. 因为其包含了很多参数, 其中常用的参数有这些:

- ARCHS -> 声明设备的架构, 目前iOS都为arm64
- TARGET -> 构建时对应的iOS版本, 默认为7.0, 可以改到10.0或按需修改
- THEOS_DEVICE_IP  -> 安装设备的ip, 一般情况下为127.0.0.1
- THEOS_DEVICE_PORT  -> 安装设备的端口, 一般情况下为2222 
- xxx\_FRAMEWORKS -> 需要一起打包的系统动态库 (这里的xxx为Tweak项目名称)
- xxx\_PRIVATE_FRAMEWORKS -> 需要一起打包的第三方动态库 (这里的xxx为Tweak项目名称)
- xxx\_FILES -> 需要打包的文件, 一般情况下是Tweak.x, **但是要打包多文件的话, 可以另起一行, 并使用```xxx_FILES += xxx.m```进行拼接**

##### **需要注意的几个问题**
1、打包Tweak项目的时候, xxx_FILES参数只需要选择```.m```文件, 不需要写头文件

2、打包Frakework的时候, 如果是私有库, 需要将库拷贝到本地的```/Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs/{系统版本}.sdk/System/Library/Frameworks/```下才能打包成功.

### 越狱环境下的常见检测点 & 反制方式

|语法层面|检测方式|检测原理|反制方式|
|---|---|---|---|
|C|open/fopen函数|通过open函数尝试打开一些越狱环境下才会存在的文件|使用fishhook对open/fopen函数进行hook, 过滤越狱黑名单(见附录1) |
|C|access函数|判断是否存在一些越狱环境下存在的特殊文件|使用fishhook对access函数进行hook, 过滤越狱黑名单(见附录2)
|C|stat函数| 判断是否存在一些越狱环境下存在的文件|使用fishhook对stat函数进行hook, 过滤越狱黑名单(见附录3)
|C|get\_env方法| 获取系统环境变量, 判断是否存在```DYLD_INSERT_LIBRARIES```环境变量|在Tweak的构造方法中对该变量进行unset|
|C|dl\_addr| 检测方法首地址, 判断方法是否被hook|在hook方法前, 先保存原函数首地址，随后hook dl\_addr函数, 返回原函数地址|
|C|dyld\_image\_count\_p| 获取动态库数量|获取动态库数量,过滤掉自身动态库以及越狱环境下的动态库之后返回剩余动态库数量|

|OC|[UIApplication canOpenURL:]|尝试打开第三方App, 如Cydia|直接返回NULL|
|OC|[NSURL canOpenURL:]|尝试打开第三方App, 如Cydia|直接返回NULL|
|OC|[NSFileManager fileExistsAtPath:]|判断文件是否存在, 原理同fopen函数|在越狱黑名单中的文件直接返回NULL|

### 反制方式Demo
#### 1、针对C函数使用fishhook进行hook, 判断操作的文件是否在黑名单中, 若在黑名单中则直接返回失败. (这里以fopen为例, 其余C函数同理)

Tweak.x文件

```
// 定义一个指针orig_fopen, 用于保存原函数.
static FILE* (*orig_fopen)(const char *filename, const char *mode);

// 定义一个dl_info, 用于在hook前保存原函数的地址.
static Dl_info fopen_info;

// 越狱文件黑名单
static char *JailbrokenPathArr[] = {"/Applications/Cydia.app","/usr/sbin/sshd", ...}

// 定义替换函数, hook fopen后过滤黑名单, 若打开的文件在黑名单中, 则返回NULL.
FILE* my_fopen(const char* filename, const char *mode) {
	NSLog(@"MT_HOOK --- fopen filename: %s, mode:%s", filename, mode);
	for (int i=0; i<sizeof(JailbrokenPathArr) / sizeof(char *); i++){
		if (strcmp(filename, JailbrokenPathArr[i]) == 0){
			NSLog(@"MT_HOOK --- %s is set to NULL, %@", filename, [NSThread callStackSymbols]);
			return NULL;
		}
	}
	FILE* ret = orig_fopen(filename, mode);
	return ret;
}

// 在构造函数中执行hook操作, 保证代码执行在最前面. 
%ctor{

	// 先保存原函数地址, 之后用fishhook进行方法替换达到hook效果
	dladdr((void*)fopen, &fopen_info);
	rebind_symbols((struct rebinding[1]){{"fopen", my_fopen, (void *)&orig_fopen}}, 1);
}

```

<br />
#### 2、针对OC层面的方法, 可以直接使用Tweak中自带的Logos进行Hook, **但同时也需要对原函数的地址进行保存**

Tewak.x文件

```
static Dl_info canOpenUrlInfo;


%ctor{
	// OC的方法需要先使用IMP获取到方法的首地址, 然后再使用dladdr进行保存
	IMP imp = class_getMethodImplementation(NSClassFromString(@"UIApplication"), @selector(canOpenURL));
	dladdr((void*)imp, &canOpenUrlInfo);
}

// 对UIApplication的canOpenURL进行Hook, 判断是否要打开cydia, 如果参数为cydia则直接返回NO
%hook UIApplication
- (BOOL)canOpenURL:(NSURL *)url{
	if ([[url scheme] isEqualToString:@"cydia"]){
		return NO;
	}
	return %orig;
}

%end
```

#### 3、对getenv进行unset操作

Tweak文件

```
%ctor{
	// 在构造函数中取消环境变量 _MSSafaMode和DYLD_INSERT_LIBRARIES
	unsetenv("_MSSafaMode");
	unsetenv("DYLD_INSERT_LIBRARIES");
}
```
